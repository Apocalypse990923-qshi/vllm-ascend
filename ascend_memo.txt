可以通过设置 `upstream` 远程仓库的方式，把原始的 vllm-ascend 仓库添加为另一个远程源，从而同步它的其他分支（比如 `v0.9.1-dev`）到你的 fork 仓库本地。

### ✅ 步骤1：添加 upstream 远程源
假设你已经 `git clone` 了你自己的 fork 仓库，例如：
git clone https://github.com/你的用户名/vllm-ascend.git
cd vllm-ascend

然后添加原始仓库为 `upstream`：
git remote add upstream https://github.com/Ascend/vllm-ascend.git
（若要添加新的upstream，需要移除原来的upstream，命令：git remote remove upstream）

验证一下远程仓库：
git remote -v

输出应该类似：
origin    https://github.com/你的用户名/vllm-ascend.git (fetch)
origin    https://github.com/你的用户名/vllm-ascend.git (push)
upstream  https://github.com/Ascend/vllm-ascend.git (fetch)
upstream  https://github.com/Ascend/vllm-ascend.git (push)

### ✅ 步骤2：获取 upstream 的所有分支
git fetch upstream
这将会把 `upstream` 仓库的所有分支拉到本地，但不会自动合并。

### ✅ 步骤3：切换到你需要的分支（比如 `v0.9.1-dev`）
git checkout -b v0.9.1-dev upstream/v0.9.1-dev
这样你就创建了一个本地分支 `v0.9.1-dev`，它的起点是 `upstream/v0.9.1-dev`。

### ✅ 步骤4（可选）：推送新分支到你自己的 fork 上
如果你想把这个分支推送到你自己的 GitHub 仓库：
git push origin v0.9.1-dev


补充：
你可以设置本地 v0.9.1-dev 去跟踪 upstream/v0.9.1-dev，以后更新就方便了：

# 设置追踪关系
git branch --set-upstream-to=upstream/v0.9.1-dev v0.9.1-dev

以后只要执行：git pull
就会把 upstream/v0.9.1-dev 的更新拉到本地 v0.9.1-dev

或者更加优雅的方式：其实你当初就可以直接这样创建分支并建立追踪：
git checkout --track -b v0.9.1-dev upstream/v0.9.1-dev

这样 Git 就知道 v0.9.1-dev 本地分支要跟踪 upstream/v0.9.1-dev 了，后面用 git pull 就能同步。

手动同步：git merge upstream/v0.9.1-dev
